Mi proyecto contiene varios archivos\+:
\begin{DoxyItemize}
\item Un programa principal (\doxylink{main_8py}{main.\+py}).
\item Un test unitario (\doxylink{test__main_8py}{test\+\_\+main.\+py}).
\item Un script (\doxylink{update__readme_8py}{update\+\_\+readme.\+py}) que ejecuta los tests y modifica el \doxylink{README_8md}{README.\+md}, con el resultado de ejecutar los test.
\item Un script básico (ci.\+yml) que ejecuta y hace commit automático con git-\/auto-\/commit-\/action.
\end{DoxyItemize}

Cuando se hago un push en main el workflow ejecuta el script en Python. El script corre los tests y modifica el \doxylink{README_8md}{README.\+md} añadiendo una de los siguientes resultados\+:

✅ Tests correctos

❌ Tests fallidos\hypertarget{md_README_autotoc_md1}{}\doxysection{\texorpdfstring{1º PARTE}{1º PARTE}}\label{md_README_autotoc_md1}

Sphinx es un framework que nos ayudará a construir la documentación de nuestros proyectos de Python. Este programa lo que hace es transformar archivos re\+ST en archivos HTML o pdf, dándoles un formato más amigable gracias al uso intero de templates.

Lo que queremos con los docstrings que escribimos es que Sphinx los entienda y logre renderizar apropiadamente un archivo en formato HTML o PDF que sea agradable de leer. Para que esto suceda debemos escribir nuestros docstrings en uno de los formatos que Sphinx entiende. En este caso he utilizado re\+Structured\+Text (re\+ST)

Para instalar Sphinx en Linux y poderlo utilizar en el proyecto de Python, he ejecutado el siguiente comando\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo\ apt\ install\ python3\ python3-\/pip\ python3-\/sphinx}

\end{DoxyCode}


Al comenzar, suponiendo que estamos en el directorio root de nuestro proyecto, nos cambiamos al directorio docs y ejecutamos el siguiente comando 
\begin{DoxyCode}{0}
\DoxyCodeLine{sphinx-\/quickstart}

\end{DoxyCode}
